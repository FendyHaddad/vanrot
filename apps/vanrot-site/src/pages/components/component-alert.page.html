Info

Neutral page message for context, setup hints, and non-blocking status. Use clear heading text so the alert remains understandable when read out of visual order.

Heads up

Success

Persistent confirmation for saved work, finished publishing, or healthy checks. Do not rely on color alone; keep the status visible in the heading and body copy.

tone.success
Published

Your changes are available in production.

 
1<vr-alert tone.success>
2<strong>Published</strong>
3<p>Your changes are available in production.</p>
4</vr-alert>

Warning

Attention state for incomplete setup, risky choices, or review-needed flows. Keep warnings actionable so the user knows whether to continue, edit, or stop.

Review needed

2<strong>Review needed</strong>
3<p>Some settings need confirmation.</p>
4</vr-alert>

Danger

Error or failed action state that should stay visible until the user can respond. Use direct recovery copy and avoid hiding danger alerts behind transient toasts.
