Convention

Role-Based File Naming

Every file name declares its role before you open it.

src/features/users/ users.page.ts users.page.html users.page.css users.i18n.ts components/ user-card.component.ts user-card.component.html user-card.component.css dialogs/ confirm-delete.dialog.ts confirm-delete.dialog.html forms/ user.form.ts user.form.html

Role suffixes supported in MVP:

.component Reusable UI pieces
.page Route-level screens
.dialog Modal / popup UI
.layout Page shells
.widget Dashboard-style blocks
.form Form-specific UI + state
Core Feature

Signals-First State

counter.component.ts

import { signal } from '@vanrot/runtime'; export class CounterComponent { count = signal(0); increment() { this.count.set(this.count() + 1); } }

counter.component.html

<p>{{ count() }}</p> <button (click)="increment()"> {{ t('counter.increase') }} </button>

counter.component.css

button { padding: 8px 12px; }
✓ Only affected text nodes update
✓ No virtual DOM
✓ CSS auto-scoped to data-vr-*
✓ i18n-ready text pattern
Developer Experience

The vr CLI

One short command for everything. Users type it constantly.

# Bootstrap npm create vanrot@latest my-app cd my-app && vr dev # Scaffold vr generate component user-card vr generate page dashboard vr generate feature users # UI ecosystem vr add button vr add dialog # Diagnostics vr doctor vr inspect counter.component vr map # Build vr build vr test

vr doctor — Zero AI needed

🔍 Nested if statements
🔍 Missing i18n keys
🔍 Raw user-facing text
🔍 SSR-unsafe browser API usage
🔍 Missing aria labels
🔍 Unscoped CSS
🔍 Missing destroy cleanup
Optional: vr explain / vr fix --ai
Pluggable providers — OpenAI, Claude, Ollama. Not required.
AI-First

Built for Humans. Predictable for AI.

AI can instantly understand:

📁 Where logic lives
📄 Where UI lives
🎨 Where styles live
🧪 Where tests live
🌍 Where translations live
🏷️ What each file represents

vr init-ai generates:

.vanrot/ai-rules.md — Use guard clauses — Use signals for state — Never put UI in TypeScript — Never put logic in HTML — Use role-based file suffixes — Use scoped CSS

vr map generates:

.vanrot/project-map.json { "pages": ["dashboard.page"], "components": ["user-card"], "routes": ["/dashboard"], "ui": ["button", "dialog"] }

Monorepo

One Root. All Packages.

vanrot/ packages/ runtime/ ← building now compiler/ vite-plugin/ cli/ ui/ router/ apps/ playground/ docs/ examples/ package.json private: true pnpm-workspace.yaml tsconfig.base.json vanrot.config.ts

Packages separate for maintainers. Unified for users.

# User sees only this: npm create vanrot@latest my-app # Generated package.json: "dependencies": { "@vanrot/runtime": "^0.1.0", "@vanrot/router": "^0.1.0" }, "devDependencies": { "@vanrot/cli": "^0.1.0", "@vanrot/vite-plugin": "^0.1.0" }
@vanrot/compiler ships as internal dep of vite-plugin and cli. Users never install it directly.
