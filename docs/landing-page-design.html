The reactive model
Only what changed.
Nothing else.

No virtual DOM. No diffing. When a signal changes, Vanrot updates the exact DOM nodes that read it — and nothing else fires.

0
Diffs per update
Signals track their own subscribers. No tree walking, no reconciliation pass.
<4kb
Runtime, gzipped
The entire reactive engine — signals, computed, effects, lifecycle — fits in 4 kilobytes.
∞
Composable anywhere
No Rules of Hooks. Create signals in loops, conditions, callbacks. They just work.
counter.component.ts 
import { signal, computed, effect } from '@vanrot/runtime';

// reactive source
const count = signal(0);

// derived — lazy, cached, never stale
const doubled = computed(() => count() * 2);

// side effect — auto-tracks deps
effect(() => {
  console.log(`doubled: ${doubled()}`);
});

count.set(3); // → "doubled: 6"
Reactive dependency graph
source signal(count) value = 3
derived computed(doubled) value = 6
DOM <span>6</span>
side effect effect() fires on change

Change propagates forward, never backward. computed() is lazy and cached — it only re-evaluates when a dependency changes, and only once per change no matter how many times it's read.
