# Conventions

## Role suffixes

Components, pages, dialogs, layouts, widgets, and forms use explicit role-based file suffixes.

Docs: /docs/conventions

## Scoped CSS

Component styles live in sibling scoped CSS files instead of global style buckets.

Docs: /docs/conventions

## Signals for state

Vanrot application state uses signals and readable helpers instead of nested imperative flow.

Docs: /docs/conventions

## Route refs

Routes use named refs and central sources of truth instead of repeated path strings.

Docs: /docs/conventions

## No UI markup in TypeScript

UI markup belongs in HTML templates, while TypeScript owns state, imports, and behavior.

Docs: /docs/conventions
