# Documentation Pages

## Vanrot

Vanrot is a small frontend framework with compiler-known templates, signals, route ownership, source-owned UI, and a documentation-first development path.

Docs: /docs

## Installation

Create or run a Vanrot app through the CLI and workspace package graph.

Docs: /docs/installation

## Project Structure

Vanrot keeps source roles explicit with TypeScript, HTML, and CSS side by side.

Docs: /docs/project-structure

## Runtime

The runtime owns signals, computed values, effects, lifecycle hooks, inputs, and mounting.

Docs: /docs/runtime

## Compiler

The compiler turns role files into mounted DOM with scoped CSS, diagnostics, source maps, control flow, child components, slots, and UI primitive lowering.

Docs: /docs/compiler

## Vite Plugin

The Vite plugin compiles Vanrot role files in dev and build, bridges diagnostics, watches sibling files, and emits source maps.

Docs: /docs/vite-plugin

## CLI

The CLI provides project creation, generation, UI primitive add flows, dev/build/test wrappers, doctor, map, config maintenance, and AI-readable local context.

Docs: /docs/cli

## Configuration

vanrot.config.ts is the typed source of truth for source roots, dev server port, UI flavor, UI style mode, and future package domains.

Docs: /docs/configuration

## Routing

The router owns route refs, nested layouts, params, query strings, redirects, guards, active links, breadcrumbs, preloading, and keepAlive.

Docs: /docs/routing

## UI October

October is Vanrot's dark-first, light-capable UI foundation with source-owned primitives, tokens, and vanrotstyles.

Docs: /docs/ui

## Theming

Vanrot themes use CSS custom properties for colors, surfaces, radius, shadows, typography, motion, and z-index layers.
